# Code of conduct

Short version: be decent, don't ship anything that violates a platform's ToS, and don't be adversarial to platforms in this repo.

## Norms

- **No ToS violations.** Every extraction path must use documented or tolerated surfaces. If in doubt, ask in an issue before submitting.
- **No security exploits.** No jailbreaks, no vulnerability-based paths, no anything that could get a user in trouble.
- **No adversarial framing in the code or docs.** We're rebalancing control, not fighting platforms. Keep the language neutral and technical.
- **Respect other contributors.** Disagree on substance; don't attack the person.
- **Respect users' data.** Nothing in this repo may exfiltrate a user's data to any third party, including the maintainers.

## Enforcement

If a contribution violates the norms, maintainers will explain the issue and give the contributor a chance to fix it. Repeated or egregious violations lead to removal.

Maintainer disputes go to a rolling maintainers' vote.

## Reporting

Open a private issue if the behaviour needs to stay confidential. Otherwise use a public issue.
